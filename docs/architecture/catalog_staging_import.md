# Staging-импорт каталога оборудования

## Назначение

Внешний каталог содержит варианты для анализа, а не готовый план закупки. Поэтому
записи из JSON, CSV и XLSX не попадают напрямую в runtime-категории ТО. Сначала
они сохраняются в `data/generated/catalog/catalog_staging.json`, проходят
валидацию и получают явный статус пользователя.

## Поток

```text
источник -> schema v1/v2 или плоская таблица -> staging -> подтверждение
         -> server/client/network -> GA -> AHP -> Pareto -> Hybrid
         -> выбранная конфигурация -> CAPEX/OPEX/энергия/NPV
```

Парсер и импорт не рассчитывают стоимость всей исходной выдачи. До Hybrid каждая
импортированная строка остаётся только альтернативой.

## Статусы

| Статус | Значение |
|---|---|
| `pending` | запись загружена и ожидает решения |
| `approved` | пользователь разрешил импорт |
| `rejected` | запись исключена пользователем |
| `blocked` | есть ошибка категории, цены или валюты |
| `imported` | запись уже перенесена в runtime ТО |

## Поддерживаемый маппинг

| Каталог | Runtime-категория | Тип компонента |
|---|---|---|
| server, rack_server, tower_server | `server` | `server` |
| prebuilt_pc, workstation, desktop, laptop | `client` | `workstation` |
| peripheral, printer, monitor | `client` | `peripheral` |
| router, switch, access_point, network_device | `network` | `network_device` |

CPU, GPU, RAM, материнские платы, накопители и корпуса намеренно блокируются.
Текущий оптимизатор выбирает готовые серверные, клиентские и сетевые устройства,
а не собирает совместимый компьютер из деталей.

## Каталог v2

Схема v2 сохраняет старые верхнеуровневые поля и добавляет:

- `source_product_id`;
- `identity` с `brand`, `model`, `mpn`, `gtin`;
- `offer` с ценой, валютой, наличием, регионом, URL и временем наблюдения;
- `field_provenance` для происхождения полей;
- `review` с исходными предупреждениями.

Staging также читает старые v1 JSON-каталоги. Это позволяет постепенно обновлять
источники без массовой миграции сохранённых примеров.

## Ограничения первого этапа

- XLSX читает первый лист и обычные значения ячеек;
- цена для расчёта должна быть положительной и заданной в RUB;
- CSV поддерживает UTF-8 и Windows-1251;
- staging не запускает Playwright и не обходит антибот-защиту;
- редактирование распознанных метрик выполняется после импорта через редактор
  компонентов или будущую расширенную форму staging.
