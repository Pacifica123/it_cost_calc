# Реализация глобальной реорганизации интерфейса

## Что изменено

Интерфейс переведён от множества верхнеуровневых аналитических вкладок к рабочим областям по предметной области:

- `Программное обеспечение`;
- `Техническое обеспечение`;
- `Редактор компонентов`;
- `Электроэнергия`;
- `NPV-анализ`;
- `Экспорт`.

Содержимое прежних вкладок `Генетический подбор`, `AHP-анализ конфигураций` и `Обоснование выбора ИТ-решений` теперь собирается внутри рабочих областей ПО и ТО. Каждая область содержит исходные позиции, эксплуатационные затраты, GA/GA+AHP, AHP и Pareto/критериальное обоснование.

## Блочная структура

Внутри ПО и ТО введены раскрываемые панели. Они позволяют скрывать тяжёлые разделы и работать с интерфейсом как с набором иерархических рабочих блоков:

- капитальные позиции;
- операционные и проектные затраты;
- матрица `ПО/ТО × CAPEX/OPEX`;
- генетический подбор и GA+AHP;
- AHP-анализ конфигураций;
- обоснование выбора ИТ-решений.

Панели размещены в `Panedwindow`, поэтому пользователь может менять размеры левой и правой областей и перераспределять место между блоками.

## Финансовая классификация

Рабочие области фиксируют следующую прикладную логику:

| Сущность | Область | Тип затрат | Обоснование |
|---|---|---|---|
| Покупка серверов, ПК, сети | ТО | CAPEX | Разовая покупка технического актива. |
| Аренда серверов, резервирование, администрирование | ТО | OPEX | Регулярная эксплуатация без покупки актива. |
| Бессрочные лицензии | ПО | CAPEX | Разовая покупка программного права/актива в модели проекта. |
| Подписки и SaaS | ПО | OPEX | Периодические платежи. |
| Миграция и тестирование | ПО/ТО | OPEX, разовые | Проектные работы внедрения, а не покупка актива. |
| Электроэнергия | ТО | OPEX | Следствие эксплуатации технической конфигурации. |

## Визуальная иерархия после адаптивной правки

После первичной реорганизации интерфейса панели визуально оставались слишком близкими к белому фону. В текущей итерации цветовая схема сделана контрастнее:

- фон приложения и рабочей области переведён в тёплый серо-бежевый тон, чтобы между панелями появились заметные «воздушные» промежутки;
- заголовки раскрываемых блоков получили отдельный оттенок, а сами блоки — явную рамку;
- тело блоков, внутренние таблицы и заголовки таблиц используют разные близкие, но различимые поверхности;
- базовый прокручиваемый контейнер теперь наследует цвет тела панели, а не общий белый фон.

Цель этой правки — приблизить рабочие области ПО/ТО к панели инструментов с отделёнными карточками, чтобы пользователь быстрее понимал, где заканчивается один смысловой блок и начинается другой.

## Экспорт

Вкладка экспорта стала контекстной:

- показывает живую сводку CAPEX/OPEX/TCO, NPV и состояния аналитики по ПО/ТО;
- формирует полный `DecisionReport`;
- поддерживает частичные markdown-экспорты с предупреждениями о неполноте контекста;
- сохраняет прежний CSV-экспорт затрат.

## Ограничения текущего этапа

Патч реорганизует интерфейс и маршрутизацию существующих аналитических панелей. Он не меняет сами критерии, веса, демо-данные и предметные метрики. Это оставлено для следующей крупной работы, чтобы не смешивать UX-реформу с методологической переработкой данных.

## Исправление сброса фоновых цветов на Windows

После внедрения контрастной схемы обнаружено, что на Windows часть `ttk`-контейнеров могла сначала отрисоваться в нужных бежевых тонах, а затем после полной инициализации окна вернуться к стандартной белой поверхности. Исправление закрепляет палитру не только через `ttk.Style`, но и через прямые фоновые параметры ключевых Tk-контейнеров.

Что изменено дополнительно:

- заголовок и тело `CollapsiblePanel` переведены на классические `tk.Frame`/`tk.Label` с явным `background`, чтобы раскрываемые блоки не зависели от поведения платформенной темы;
- для вложенных аналитических панелей добавлен повторный проход `force_panel_backgrounds`, который после сборки дочерних виджетов назначает безопасные panel-стили `ttk`-элементам;
- после создания всех вкладок приложение повторно стабилизирует тему сразу, через `after_idle` и через короткую задержку, чтобы поздняя перерисовка Windows не сбрасывала поверхность к белому;
- компактные табличные секции используют `tk.LabelFrame` с явным цветом тела блока и рамкой.

Эта правка не меняет расчётную логику и предназначена только для устойчивого визуального разделения уровней: окно приложения → рабочая область → блок → тело блока → таблица.
