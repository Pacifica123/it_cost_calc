# Qt settings, responsive and performance pass

Патч вводит слой настроек перед переключением Qt как основного UI.

## Что добавлено

- сохранение темы, масштаба, стартового раздела и режима расширенных параметров;
- настройки хранятся в `data/generated/qt_ui_settings.json`;
- корневое меню и верхняя панель получают compact-режим при узком окне;
- добавлены горячие клавиши для демо, основного действия и переходов workflow;
- старт Qt shell остаётся lazy: тяжёлые экраны создаются при первом открытии;
- `QtMainWindow.startup_ms` фиксирует базовую метрику создания окна.

## Горячие клавиши

- `Ctrl+L` — загрузить демо;
- `Ctrl+Enter` — главное действие текущего шага;
- `Alt+Right` — следующий шаг workflow;
- `Alt+Left` — предыдущий шаг workflow;
- `F1` — помощь текущего экрана;
- `Esc` — выйти из настроек.

## Риски

- настройки не должны превращаться в отдельное сложное приложение;
- responsive-режим меняет только оболочку, не трогая бизнес-экраны;
- масштаб меняет шрифт приложения, но не переписывает QSS.

## Проверки

- чистая загрузка и сохранение настроек;
- нормализация некорректных значений;
- сохранение коротких видимых текстов;
- отсутствие eager-создания всех экранов в shell-контракте.
