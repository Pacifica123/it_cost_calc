# Локальный запуск и разработка

## Базовый запуск

Установить зависимости:

```bash
pip install -r requirements/base.txt
```

Запустить приложение:

```bash
python scripts/run_app.py
```

## Загрузка демонстрационных данных

По умолчанию приложение стартует без автоматической подгрузки демонстрационного набора.
Если нужен демонстрационный пример, его можно загрузить вручную:

```bash
python scripts/load_demo_data.py
```

Либо нажать кнопку **«Загрузить демо-данные»** внутри GUI.

## Запуск тестов

```bash
pytest
```

## Что важно помнить при разработке

- основные пользовательские сценарии опираются на прикладные сервисы и use case-модули;
- часть совместимых wrapper-слоёв сохранена ради поддержки переходных участков кода;
- AHP, анализ важности критериев и динамическая инфраструктурная вкладка требуют особенно аккуратных изменений;
- `data/generated/` — это runtime-данные, а не место для хранения исходных эталонных материалов.

## Рекомендации по дальнейшей работе

1. Не смешивать инженерные документы и дипломные исходники.
2. Не возвращать debug-артефакты в корень репозитория.
3. Новые алгоритмические модули сразу оформлять через `domain` и `application`, а не прятать в UI.
4. Временные исследовательские скрипты либо сразу убирать, либо складывать в `tools/` с понятным назначением.

## Обновление каталога оборудования

Пока GUI не запускает парсер напрямую, для подготовки нормализованного каталога используется отдельный скрипт:

```bash
python scripts/update_equipment_catalog.py --mode examples
```

Результат сохраняется в `data/generated/catalog/equipment_catalog.json`.
