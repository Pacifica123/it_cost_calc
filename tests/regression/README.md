# Регрессионные тесты

Здесь лежат тесты, которые фиксируют ожидаемое поведение ключевых модулей на стабильных эталонных кейсах.

Сейчас сюда вынесены два главных сценария:
- AHP-анализ конфигураций на примере из `data/examples/ahp/test_confs.json`;
- анализ важности критериев на встроенном кейсе выбора системы бюджетирования.

Идея этих тестов простая: после дальнейших изменений проект должен не только запускаться, но и выдавать тот же осмысленный результат на уже согласованных примерах.
