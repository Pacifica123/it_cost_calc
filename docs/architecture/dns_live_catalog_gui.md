# Управляемый сбор каталога DNS из GUI

## Назначение

Экран `Каталог` позволяет пользователю запустить ограниченный сбор публичных карточек DNS и сразу передать результат в существующий staging. GUI не импортирует Playwright и не разбирает HTML: он запускает самостоятельный CLI-процесс через `QProcess`, показывает его журнал и принимает только готовый catalog v2.

```text
Qt dialog
  -> отдельный CLI dns-live
  -> видимый persistent Chromium
  -> search/*.html + products/*.html + snapshot_manifest.json
  -> канонический dns-snapshot importer
  -> equipment_catalog.json schema v2
  -> staging review
  -> явное подтверждение и импорт в ТО
```

## Пользовательский сценарий

1. Открыть раздел `Каталог` и нажать `Собрать из DNS`.
2. Выбрать роутеры, готовые ПК и/или серверы.
3. Задать лимит карточек на категорию, общий таймаут и текстовую метку региона.
4. Оставить `Показывать браузер` включённым.
5. На первой странице при необходимости выбрать регион, принять cookies или пройти обычную проверку сайта.
6. Следить за журналом; при необходимости нажать `Остановить`.
7. После успешного завершения нажать `Загрузить в staging`.
8. Проверить warnings, исправить поля, подтвердить и импортировать нужные варианты в ТО.

## Ограничения и воспроизводимость

- на категорию разрешено от 1 до 50 карточек;
- общий таймаут ограничен диапазоном 30–1800 секунд;
- между карточками есть пауза;
- ссылки принимаются только с HTTPS-хостов `dns-shop.ru`;
- автоматический обход challenge не реализуется;
- профиль Chromium сохраняется в `data/generated/catalog/dns_browser_profile/`;
- каждый запуск сохраняется в `data/generated/catalog/dns_runs/<timestamp>/`;
- `search/` помогает диагностировать отсутствие ссылок, а `products/` можно повторно нормализовать без сети.

Если сайт вернул challenge или региональную заглушку, сбор завершится без карточек. Пользователь может изучить сохранённый search HTML, повторить запуск с видимым браузером и увеличить общий таймаут.

## Отмена и ошибки

Кнопка остановки посылает CLI процессу `SIGTERM`. CLI преобразует его в управляемую отмену, закрывает browser context и возвращает код 130. Если процесс не завершился за три секунды, GUI выполняет принудительную остановку.

Ошибки установки Playwright/Chromium, навигации и пустой выдачи выводятся в журнале диалога. Неуспешный результат не загружается в staging.

## Граница ответственности

`tools/catalog_parser/` остаётся отдельным инструментом. GUI отвечает только за параметры задания, состояние процесса и передачу готового файла в `CatalogStagingService`. Сетевая работа не выполняется в главном Qt-потоке и не влияет на расчётные сервисы приложения.
