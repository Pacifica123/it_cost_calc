# Исследование: AHP-анализ конфигураций

## Зачем нужен этот модуль

AHP-модуль предназначен для выбора оптимальной конфигурации оборудования из множества альтернатив, когда решение нельзя свести только к одной цифре стоимости.

Важная идея здесь такая:
- сначала отсеиваются варианты, которые не проходят по жёстким ограничениям;
- затем оставшиеся конфигурации сравниваются по мягким критериям;
- веса критериев определяются на основе экспертных матриц попарных сравнений.

## Что подаётся на вход

На вход AHP-pipeline подаются:

- список конфигураций;
- список мягких критериев;
- матрицы попарных сравнений критериев от экспертов;
- словарь жёстких ограничений.

В коде это сосредоточено в:
- `src/domain/decision/ahp/configuration_selector.py`
- `src/domain/decision/ahp/pipeline.py`

## Что происходит внутри

1. Каждая конфигурация агрегируется в набор сводных показателей.
2. Применяются жёсткие ограничения.
3. Для критериев агрегируются экспертные оценки.
4. Для каждой группы альтернатив строятся матрицы попарных сравнений.
5. Вычисляются приоритеты и итоговые веса.
6. Строится ranking альтернатив и выбирается победитель.

## Что получается на выходе

На выходе формируется отчёт, в котором есть:
- список прошедших альтернатив;
- детали отфильтрованных конфигураций;
- веса критериев;
- локальные веса альтернатив по каждому критерию;
- итоговое ранжирование;
- выбранная лучшая конфигурация.
