# Идея: гибридная итоговая оценка ГА и AHP

Статус документа: **идея для будущего развития**.  
Документ описывает методическую концепцию гибридной оценки и критерии её релевантности. Это не техническое задание и не план реализации по шагам.

## Зачем нужна гибридная оценка

После независимого сравнения ГА и AHP может возникнуть практический вопрос: какой вариант считать итогово лучшим, если методы немного расходятся?

Например:

```text
ГА считает лучшим вариант A.
AHP считает лучшим вариант B.
Оба варианта находятся в верхнем слое и отличаются не радикально.
```

В такой ситуации полезна гибридная оценка. Она не должна скрывать расхождения между методами. Её задача — дать дополнительный итоговый ориентир, который учитывает оба взгляда:

- поисковую оценку ГА;
- аналитическую многокритериальную оценку AHP.

## Важная граница идеи

Гибридная оценка не должна использоваться как способ замаскировать методический конфликт.

Если ГА и AHP радикально расходятся, сначала нужно показать пользователю это расхождение. Только после этого можно давать гибридный результат как дополнительный компромиссный показатель.

Корректная логика:

```text
сначала независимые оценки → затем анализ согласованности → затем гибридная оценка
```

Некорректная логика:

```text
методы сильно спорят → смешать числа → показать один красивый победитель
```

## Основная идея расчёта

Для каждой конфигурации есть две независимые оценки:

```text
G_i — оценка конфигурации i по ГА
A_i — оценка конфигурации i по AHP
```

Чтобы их можно было объединять, они должны быть приведены к общей шкале и одной направленности:

```text
чем больше значение, тем лучше вариант
```

После приведения можно считать гибридную оценку как взвешенное среднее:

```text
H_i = λ × Ĝ_i + (1 - λ) × Â_i
```

Где:

- `H_i` — гибридная итоговая оценка конфигурации;
- `Ĝ_i` — приведённая оценка ГА;
- `Â_i` — приведённая оценка AHP;
- `λ` — вес доверия к ГА;
- `1 - λ` — вес доверия к AHP.

При нейтральном сценарии можно использовать:

```text
λ = 0.5
```

Тогда ГА и AHP влияют на итог одинаково.

## Почему нужно предварительное приведение оценок

GA-score и AHP-score могут иметь разную природу.

GA-score обычно отражает fitness-функцию поискового алгоритма. AHP-score часто является приоритетом альтернативы внутри конкретного набора и может быть нормирован так, что сумма оценок равна `1`.

Если смешивать такие числа напрямую, можно получить некорректный результат. Поэтому перед объединением нужен общий масштаб.

Идейная схема приведения:

1. фиксируется общий пул конфигураций;
2. для каждой конфигурации считаются независимые `G_i` и `A_i`;
3. обе оценки приводятся к шкале `0..1` внутри этого пула;
4. после этого считается гибридная оценка `H_i`.

Пример приведения через min-max:

```text
Ĝ_i = (G_i - min(G)) / (max(G) - min(G))
Â_i = (A_i - min(A)) / (max(A) - min(A))
```

Если все значения по одному методу одинаковые, этот метод не различает варианты. В таком случае его приведённые значения можно считать равными для всех конфигураций, а акцент в объяснении переносить на другой метод и на сам факт низкой различающей способности.

## Почему взвешенное среднее релевантно

Взвешенное среднее подходит для гибридной оценки, потому что оно обладает несколькими важными свойствами.

### 1. Понятность

Пользователю легко объяснить смысл формулы:

```text
итоговая оценка частично учитывает мнение ГА и частично мнение AHP
```

Это лучше, чем вводить сложную непрозрачную формулу, которую трудно защитить на дипломе и трудно объяснить пользователю.

### 2. Ограниченность результата

Если `Ĝ_i` и `Â_i` находятся в диапазоне `0..1`, то при `0 ≤ λ ≤ 1` гибридная оценка тоже остаётся в диапазоне `0..1`.

Это делает результат стабильным и удобным для отображения.

### 3. Монотонность

Если одна конфигурация лучше другой и по ГА, и по AHP, то её гибридная оценка тоже будет не ниже.

То есть если:

```text
Ĝ_X ≥ Ĝ_Y
Â_X ≥ Â_Y
```

то при `0 ≤ λ ≤ 1` выполняется:

```text
H_X ≥ H_Y
```

Это важное свойство: гибридная формула не должна случайно понижать вариант, который сильнее сразу по двум независимым оценкам.

### 4. Управляемость методической позиции

Параметр `λ` позволяет честно зафиксировать, какой метод важнее в конкретной интерпретации.

Возможные режимы:

| Значение `λ` | Смысл |
|---:|---|
| `0.5` | Нейтральный компромисс между ГА и AHP |
| `> 0.5` | Больше доверия поисковой fitness-оценке ГА |
| `< 0.5` | Больше доверия экспертно-аналитической оценке AHP |

Для дипломного сценария наиболее безопасен нейтральный вариант `λ = 0.5`, потому что он не объявляет один метод главным без дополнительного обоснования.

### 5. Сохранение объяснимости

Гибридный балл можно разложить на две части:

```text
вклад ГА = λ × Ĝ_i
вклад AHP = (1 - λ) × Â_i
```

Благодаря этому пользователь может увидеть, почему вариант получил итоговую оценку: он силён по ГА, по AHP или сбалансирован по обоим методам.

## Почему не стоит добавлять бонус за совпадение прямо в формулу

Можно захотеть добавить в итоговую оценку бонус за согласие методов. Например, награждать вариант, если ГА и AHP дают ему похожие оценки.

Но это опасно. Такой бонус может нарушить монотонность: вариант, который чуть хуже по обоим методам, может получить преимущество только потому, что оценки по нему совпали.

Поэтому согласованность лучше показывать отдельной диагностикой, а не включать в сам гибридный score.

Идейная схема:

```text
H_i — итоговая гибридная оценка качества
Δ_i — показатель расхождения между ГА и AHP
```

Где:

```text
Δ_i = |Ĝ_i - Â_i|
```

Чем меньше `Δ_i`, тем согласованнее методы оценили конкретный вариант.

## Как показывать результат пользователю

В будущем интерфейс мог бы показывать не один скрытый итог, а понятную таблицу:

| Вариант | GA score | AHP score | Гибридный score | Расхождение | Комментарий |
|---|---:|---:|---:|---:|---|
| A | 0.95 | 0.88 | 0.915 | 0.07 | Сильный и согласованный вариант |
| B | 0.90 | 0.93 | 0.915 | 0.03 | Компромиссный лидер |
| C | 0.80 | 0.55 | 0.675 | 0.25 | Методы заметно расходятся |

Такой формат не заставляет пользователя выбирать, кому верить. Он показывает три слоя смысла:

1. что думает ГА;
2. что думает AHP;
3. какой вариант получается как компромисс.

## Как использовать гибридную оценку вместе с анализом согласованности

Гибридная оценка особенно полезна при умеренном расхождении методов.

| Ситуация | Как трактовать гибридную оценку |
|---|---|
| ГА и AHP почти согласны | Гибридный score подтверждает общий выбор |
| Есть небольшая перестановка top-2/top-3 | Гибридный score помогает выбрать компромисс |
| Методы выбирают разные группы лидеров | Гибридный score показывается с предупреждением |
| Один метод не различает варианты | Гибридный score опирается главным образом на второй метод |

То есть гибридная оценка не отменяет диагностику. Она работает вместе с ней.

## Короткая формулировка идеи

> Гибридная оценка должна объединять независимый GA-score и независимый AHP-score только после приведения к общей шкале. Наиболее прозрачная методика — взвешенное среднее приведённых оценок, потому что она ограничена, монотонна, объяснима и позволяет явно задавать баланс доверия между поисковой и аналитической оценкой.

## Польза для проекта

Такая фича усиливает проект в трёх направлениях:

1. **Пользовательская ясность** — пользователь видит не конфликт методов, а структурированное сравнение.
2. **Методическая аккуратность** — ГА и AHP не смешиваются хаотично, а приводятся к общей шкале перед объединением.
3. **Защитная ценность** — на защите можно объяснить, почему итоговый выбор не является произвольным: он строится как компромисс двух независимых оценок с сохранением монотонности и прозрачности.

## Что не является целью этой идеи

Гибридная оценка не должна заменять ГА, AHP или анализ расхождений. Она должна быть дополнительным итоговым слоем, который появляется только после того, как пользователь увидел независимые оценки и степень их согласованности.
