# Qt component editor screen

## Назначение

Патч переносит первый полноценный бизнес-экран в новый Qt-контур: редактор компонентов решения. Tkinter-интерфейс не переключается и остаётся рабочим, но раздел `Компоненты` в Qt shell теперь открывает реальный CRUD-экран, а не заглушку.

## UX-правила

- видимые подписи короткие;
- основной маршрут: `Выбрать → Изменить → Сохранить`;
- таблица показывает только ключевые поля;
- редкие действия находятся в `Ещё`;
- профильные метрики скрыты в раскрываемом блоке;
- описание, допущения и строгий режим скрыты отдельно;
- длинные пояснения доступны через tooltip.

## Основные файлы

| Файл | Назначение |
|---|---|
| `src/ui_qt/screens/component_editor.py` | Qt-экран редактора компонентов |
| `src/ui_qt/presenters/component_editor_presenter.py` | GUI-free presenter для CRUD и нормализации |
| `src/ui_qt/models/solution_component_table_model.py` | таблица сохранённых компонентов |
| `src/ui_qt/main_window.py` | подключение экрана к root-разделу `Компоненты` |

## Граница с логикой

Экран не реализует расчёты и не обращается к Tkinter. CRUD идёт через `SolutionComponentRuntimeService`, нормализация — через `SolutionComponentNormalizationService`, хранилище — через текущий `JsonEntityRepository` из Qt adapter layer.

## Проверки

- создание payload из формы;
- запрет сохранения без названия;
- создание, редактирование и удаление компонента;
- проверка коротких заголовков таблицы;
- общий scan плотности текста.
