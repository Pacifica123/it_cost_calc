# Qt NPV screen

Патч `qt-npv-screen` переносит раздел `NPV-анализ` в новый Qt-контур без переписывания финансовой логики.

## Что добавлено

- `src/ui_qt/screens/npv.py` — компактный экран NPV.
- `src/ui_qt/presenters/npv_presenter.py` — presenter без Qt/Tkinter зависимостей.
- `QtAppPresenter` получил методы подготовки TCO-базы и построения NPV-отчёта.
- Раздел `NPV` в главном Qt shell теперь открывает рабочий экран.

## UX-правила

Экран следует контракту миграции:

- один экран — одна финансовая задача;
- видимые подписи короткие;
- подробные потоки и TCO-база скрыты в раскрываемых блоках;
- главный action один: `Рассчитать`;
- подстановка текущих затрат вынесена в отдельное действие `Из TCO`.

## Финансовая связка

Экран может работать в двух режимах:

1. Ручной ввод инвестиций, ставки и потоков.
2. Подстановка базы из текущих CAPEX/OPEX/электроэнергии через `PrepareCostSummaryUseCase` и `BuildNpvReportUseCase.prepare_from_totals`.

Сама формула NPV остаётся в `domain.finance.npv`, отчёт строится через существующий `NPVReportService`.

## Известные ограничения

- Визуальный график из старого Tkinter-экрана пока не переносится.
- Денежные потоки вводятся строкой через запятую.
- Общий polish форм будет выполнен отдельным проходом после переноса нескольких экранов.
