# Демонстрационные сценарии

Этот раздел содержит короткие материалы для показа проекта на консультациях, промежуточных защитах и защите обновлений перед научным руководителем.

## Материалы

- `ga_ahp_defense_guide.md` — актуальный сценарий демонстрации GA, AHP, Pareto и Hybrid внутри рабочих областей ПО/ТО.
- `demo_data_contract.md` — контракт ролей demo/educational/regression-данных и контрольный сценарий после этапа 8.
- `solution_component_editor_defense_guide.md` — финальный сценарий защиты редактора `SolutionComponent` после C9.

## Как использовать

Перед показом проекта достаточно открыть нужный сценарий и пройтись по нему как по чек-листу:

1. что подготовить перед запуском;
2. какие вкладки показать;
3. что говорить на каждом шаге;
4. как объяснить результат простыми словами;
5. как отвечать на вопросы научного руководителя.

## Контроль демо-данных после этапа 8

Сквозной демонстрационный набор проверяется без `pytest`:

```bash
python -B scripts/run_demo_control_scenario.py --check-only
```

Проверка подтверждает, что fixture-строки имеют `scope` и `component_type`, формируют общий пул `CandidateConfiguration`, получают TCO и попадают в `DecisionReport`.

## Контроль финального C9-сценария редактора компонентов

Для показа именно `SolutionComponent` используется отдельная fixture:

```bash
PYTHONDONTWRITEBYTECODE=1 python -B scripts/run_solution_component_c9_demo.py
```

Проверка подтверждает четыре защитных примера: mixed-комплект, ПО-подписку, периферию без клиентского места и черновик с warnings.

## Итоговый демонстрационный маршрут после roadmap

Финальный демонстрационный маршрут удобно показывать так:

1. загрузить demo-набор;
2. показать CAPEX/OPEX и энергетические параметры;
3. объяснить, что записи получают `scope` и `component_type`;
4. запустить или показать цепочку GA → общий пул → AHP/Pareto → Hybrid;
5. открыть NPV как финансовое продолжение рассчитанных затрат;
6. открыть `SolutionComponent`-сценарий и показать, что нестандартные компоненты идут через ту же модель;
7. сформировать `DecisionReport` в экспорте;
8. показать, что отчёт содержит компоненты, альтернативы, стоимость, аналитику, предупреждения и объяснение выбора.

Этот маршрут хорошо подходит для защиты, потому что демонстрирует не отдельные кнопки, а цельную цепочку принятия решения.
