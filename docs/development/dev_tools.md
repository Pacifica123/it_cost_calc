# Инструменты разработки

В проекте используется минимальный набор dev-инструментов:
- **pytest** для тестов;
- **ruff** как основной линтер;
- **black** как форматтер;
- единая конфигурация в `pyproject.toml`.

## Установка

Из корня репозитория:

```bash
pip install -r requirements/dev.txt
```

## Основные команды

Проверка тестов:

```bash
pytest
```

Проверка стиля и ошибок:

```bash
ruff check src tests scripts
```

Автоформатирование:

```bash
black src tests scripts
```

Комплексная локальная проверка перед коммитом:

```bash
ruff check src tests scripts && black --check src tests scripts && pytest
```

## Pre-commit

В репозиторий добавлен `.pre-commit-config.yaml`.

Подключение:

```bash
pip install pre-commit
pre-commit install
```

После этого перед коммитом будут автоматически запускаться базовые проверки стиля.

## Область строгих проверок

Линтер и форматтер ориентированы в первую очередь на **активный код проекта**:
- `src/`
- `tests/`
- `scripts/`

Исторические и legacy-артефакты вроде старых parser-экспериментов не рассматриваются как основной контур ежедневной разработки и поэтому исключены из строгих проверок.
