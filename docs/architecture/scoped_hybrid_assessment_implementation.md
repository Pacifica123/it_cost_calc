# Реализация scoped hybrid assessment поверх общего пула

## Что изменено

Патч переводит гибридную итоговую оценку на более честную схему работы внутри рабочих областей `ПО` и `ТО`:

`GA формирует общий пул → AHP ранжирует тот же пул → Pareto анализирует тот же пул → Hybrid собирает последнюю витрину`.

Гибридная панель больше не выглядит как ещё один запуск `GA + AHP`. Она не запускает GA повторно и не подменяет AHP. Вместо этого она ждёт явной передачи общего пула и последних результатов соседних блоков.

## Зачем это нужно

До этого даже после выноса гибридной панели оставался методический риск: пользователь мог нажать расчёт гибрида и получить новый сценарий `GA→AHP`, то есть фактически вернуть старую схему смешанного блока. Для защиты и для дальнейшего развития проекта важнее показать простую цепочку ответственности:

- `GA` ищет допустимые кандидатные конфигурации;
- `AHP` самостоятельно ранжирует переданный набор;
- `Pareto` самостоятельно показывает доминирование и компромиссы;
- `Hybrid` только согласует уже полученные результаты.

## Как это работает в UI

1. Пользователь запускает `GA`.
2. Пользователь нажимает `Передать top-N в общий пул`.
3. Рабочая область передаёт этот пул в `AHP`, `Pareto` и `Hybrid`.
4. Пользователь запускает самостоятельный расчёт `AHP`.
5. Пользователь запускает самостоятельный расчёт `Pareto`.
6. Пользователь открывает `Hybrid` и нажимает `Обновить гибридную витрину`.

Если AHP ещё не рассчитан, гибридная панель честно пишет, что итоговую оценку считать нельзя. Это сделано намеренно: гибрид не должен скрыто пересчитывать AHP внутри себя.

## Методика базовой гибридной оценки

В числовой `hybrid_score` включаются только две нормированные компоненты:

`hybrid_score = λ × normalized_ga_score + (1 - λ) × normalized_ahp_score`

Где:

- `normalized_ga_score` — GA-score кандидата, нормированный внутри общего пула;
- `normalized_ahp_score` — AHP-score кандидата, нормированный по последнему AHP-результату;
- `λ` — коэффициент доверия к GA в диапазоне `0..1`.

Pareto не включается как скрытый вес. Он отображается как статус:

- `недоминируемая`;
- `доминируемая`;
- `нет данных`.

Такой вариант намеренно проще и безопаснее: Pareto пока работает как фильтр компромиссов и предупреждение, а не как ещё один ранжирующий критерий с неочевидным весом.

## Состояние методов внутри workspace

В `SolutionAreaWorkspaceTab` добавлено минимальное состояние:

| Поле | Назначение |
|---|---|
| `candidate_pool_snapshot` | последний общий пул текущей области |
| `ahp_result` | последний самостоятельный AHP-отчёт |
| `pareto_result` | последний самостоятельный Pareto/criteria-importance отчёт |
| `hybrid_result` | место для последнего гибридного результата |

При передаче нового пула AHP/Pareto/Hybrid считаются устаревшими и сбрасываются. Это базовый вариант dirty-логики без event bus, БД и истории запусков.

## Основные файлы

| Файл | Роль |
|---|---|
| `src/application/services/hybrid_decision_assessment_service.py` | чистый сервис гибридной оценки по общему пулу, AHP-отчёту и Pareto-отчёту |
| `src/ui/tabs/hybrid_decision_assessment_tab.py` | новая гибридная панель без повторного запуска GA/AHP |
| `src/ui/tabs/solution_area_workspace_tab.py` | минимальное состояние методов и передача результатов между соседними блоками |
| `src/ui/tabs/ahp_analysis/tab.py` | callback последнего AHP-результата в workspace |
| `src/ui/tabs/criteria_importance/tab.py` | callback последнего Pareto-результата в workspace |
| `tests/unit/test_hybrid_decision_assessment_service.py` | unit-проверки базовой гибридной методики |

## Ограничения текущего шага

- Состояние не сохраняется между запусками приложения.
- Гибридная панель пока не экспортирует отдельный раздел в общий DecisionReport.
- Нет автоматической проверки, что пользователь вручную не изменил AHP-конфигурации после загрузки пула.
- Pareto-статус не влияет на числовой score, а только добавляет предупреждение.
- Сложной истории запусков и сравнения нескольких пулов пока нет.

## Почему это не переусложняет проект

Патч не вводит БД, event bus, отдельные фоновые задачи или новый сложный алгоритм. Он только добавляет недостающую проводку состояния между уже существующими блоками и делает гибридную панель честной витриной поверх результатов, а не скрытым комбайном `GA + AHP`.
